  The real bug: %q double quotes inside onclick="..." attributes

  The rohanthewiz/element builder writes attribute values verbatim — no HTML
  escaping. So fmt.Sprintf("connectCamera(%q,...)", ...) produced:

  <div onclick="connectCamera("Main","192.168.5.90","52381")">

  The browser parsed onclick="connectCamera(" and then treated the rest as
  garbage HTML attributes. Every server-rendered camera card button (connect,
  reconnect, edit, delete) was broken on page load.

  This explains why:
  - Fresh start works: No saved cameras → user fills the form → JS handles
  everything with single-quoted strings → no broken HTML
  - After restart fails: Saved cameras are server-rendered with broken %q double
   quotes → clicking cameras does nothing → can't connect → all movement
  controls return "not connected"

  Fix: Replaced all %q formatting in onclick attributes with single-quoted JS
  strings ('%s') and added a jsEscape helper that backslash-escapes single
  quotes and backslashes in labels/IPs.

     Migrate JSON config storage to DuckDB

     Context

     Camera and preset configs stored in JSON files (cameras.json, presets.json)
      are getting mangled during server restarts. Replacing with DuckDB via
     github.com/duckdb/duckdb-go/v2 for reliable persistence. DuckDB uses the
     standard database/sql interface.

     Approach

     Create a single storage package with one *sql.DB wrapping both tables.
     Delete the old cameras/ and presets/ packages. Keep the same method
     signatures so handler changes are minimal.

     Steps

     1. Create storage/storage.go

     - Define DB struct wrapping *sql.DB, plus Camera and Preset types (moved
     from old packages)
     - Open(dbPath string) (*DB, error) — opens DuckDB, creates tables, seeds 6
     default presets if empty
     - Methods matching current API: AllCameras, UpsertCamera, UpdateCamera,
     RemoveCamera, AllPresets, UpdatePresetLabel
     - Close() error
     - Tables:
     CREATE TABLE IF NOT EXISTS cameras (label TEXT PRIMARY KEY, ip TEXT NOT
     NULL, port INTEGER NOT NULL);
     CREATE TABLE IF NOT EXISTS presets (number INTEGER PRIMARY KEY, label TEXT
     NOT NULL);
     - UpsertCamera uses INSERT OR REPLACE INTO
     - UpdateCamera uses a transaction for label renames (delete old + insert
     new)
     - Default preset seeding: if presets table empty, insert "Preset 1" through
      "Preset 6"

     2. Create storage/migrate.go

     - One-time migration: if cameras.json/presets.json exist next to the DB
     file, read them, insert data, rename to .json.migrated
     - Runs inside Open() after table creation, before default seeding
     - Errors logged but not fatal

     3. Update main.go

     - Replace two store inits with storage.Open("camctl.db")
     - Pass single *storage.DB to handlers.NewApp(store)
     - Add defer store.Close()

     4. Update handlers/handlers.go

     - App struct: replace Presets *presets.Store + Cameras *cameras.Store with
     Store *storage.DB
     - NewApp(store *storage.DB) *App
     - Update all call sites: a.Cameras.All() → a.Store.AllCameras(), etc.
     - Handle new error returns from AllCameras()/AllPresets()

     5. Update views/views.go

     - Change PageData.Presets type from []presets.Preset to []storage.Preset
     - Update renderPresets parameter type
     - Fields (Number, Label) are identical — no body changes

     6. Delete old packages

     - Remove cameras/cameras.go and presets/presets.go

     7. Dependencies & cleanup

     - go get github.com/duckdb/duckdb-go/v2 + go mod tidy
     - Add camctl.db, *.json.migrated to .gitignore

     Files to modify

     - storage/storage.go — new
     - storage/migrate.go — new
     - main.go
     - handlers/handlers.go
     - views/views.go
     - cameras/cameras.go — delete
     - presets/presets.go — delete
     - go.mod / go.sum
     - .gitignore

     Notes

     - DuckDB requires CGO (enabled by default on macOS)
     - Binary size increases ~50MB (embedded DuckDB engine)
     - All* methods now return ([]T, error) instead of []T

     1. go build ./... compiles cleanly
